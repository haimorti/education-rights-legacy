# מחולל מדריכי ה-PDF — "שיקום מקצועי לסטודנטים"

מנגנון להפקת מדריכי **PDF נאמנים לאתר** מתוך תוכן הזכאויות, להעברה למבוטחים ב**אימייל ובפגישה פנים-אל-פנים** (כשאי אפשר להטמיע אתר חיצוני). ראו את מסמך המוצר המלא: [`SPEC.md`](./SPEC.md).

> תקציר התוצר: לכל מדריך **שני קבצים** — אחד **דסקטופ** (עמודי A4, רקע לבן) ואחד **מובייל** (עמוד אחד ארוך רציף, רקע בהיר). שניהם שומרים על העיצוב, הצבעים, הפונטים והרכיבים של האתר.

---

## ארכיטקטורה (Pipeline)

```
docs/*.md  ─┐
            ├─►  מחולל (Python)  ─►  HTML מעוצב  ─►  render.js (Chromium)  ─►  pdf/*.pdf
base.css  ──┘        ▲
fonts/  ────────────-┘  (Rubik + Heebo מוטמעים)
```

- **מקור-אמת לתוכן:** `../docs/*.md`.
- **עיצוב:** קובץ CSS אחד — `base.css` — מוטמע בכל HTML בזמן הבנייה (שינוי גלובלי = מקום אחד).
- **רינדור:** `render.js` פותח את ה-HTML ב-Chromium ומפיק שתי גרסאות:
  - **desktop** → A4 מרובה-עמודים (חד גם על מסך מחשב).
  - **mobile** → עמוד אחד ארוך (חד ממילא בנייד).
- **פונטים:** Rubik / Heebo, מצורפים ב-`fonts/` ומוטמעים — אין תלות ברשת.

## מבנה התיקייה

```
pdf-generator/
├── base.css            # מערכת העיצוב המלאה + כללי שבירת עמודים (משותף לכל העמודים)
├── generate.py         # מחולל גנרי מ-Markdown (פולבק לעמודים שטרם נבנו ייעודית)
├── pages/
│   ├── app01.py        # בנייה ייעודית — עמוד "הגשת בקשה" (תהליך)
│   └── app02.py        # בנייה ייעודית — עמוד "דמי שיקום"
├── render.js           # HTML → PDF (דסקטופ A4 + מובייל ארוך) דרך Chromium
├── fonts/              # Rubik + Heebo (woff2) + fonts.local.css
├── build.sh            # סקריפט בנייה נוח
├── package.json        # תלות יחידה: playwright-core
├── SPEC.md             # מפרט המוצר המלא
└── _html/              # HTML ביניים (נוצר אוטומטית, ב-.gitignore)

../pdf/                 # התוצרים הסופיים (PDF) — נשמרים בריפו
```

## דרישות והתקנה

```bash
cd pdf-generator
npm install                       # מתקין playwright-core (פעם אחת)
# Chromium: ברירת המחדל מחפשת את הדפדפן של playwright בסביבה.
# אם צריך, הצביעו על דפדפן מקומי:
export CHROME_PATH=/path/to/chromium
```
דרוש גם `python3` (ללא תלויות חיצוניות) ו-`node`.

## בנייה

```bash
./build.sh 02            # בונה עמוד יחיד (דמי שיקום) → ../pdf/02-*.{desktop,mobile}.pdf
./build.sh 01 02         # כמה עמודים
./build.sh all           # כל העמודים
```

## איך מוסיפים עמוד זכאות חדש (בנייה ייעודית)

עמודי הזכאות **אינם** רינדור שטוח מ-Markdown — הם משחזרים את **רכיבי האתר המקוריים** (כרטיסים ממוספרים, תת-כרטיסים זה-לצד-זה, הדגשות/pills, תרחישים ומדרגות צבעוניים, תוויות נכה כללי/נפגע עבודה וכו'), בדיוק כפי שנעשה בעמוד התהליך ובדמי שיקום.

תהליך הוספת עמוד (למשל שכר לימוד):
1. קראו את ה-component המקורי של אותה זכאות באתר (`src/components/<benefit>/<benefit>-accordion.tsx`).
2. העתיקו את `pages/app02.py` כתבנית ל-`pages/app03.py` והתאימו: hero (אייקון/כותרת/תת-כותרת), כרטיס ההקדמה (זהה לכל הזכאויות), וגוף התוכן לפי ה-component — בעזרת העוזרים הקיימים: `sechead, cond, callout, pill, twoup, tier, scenario, innerbox, linkout, checkbullets, dlabel, card, summary, important`.
3. רשמו את ה-builder ב-`build.sh` (מערך `BESPOKE`).
4. `./build.sh 03` ובדקו את שתי הגרסאות.

רכיבי עיצוב משותפים נמצאים ב-`base.css` (חפשו "benefit-page rich components"). אפשר להוסיף רכיבים חדשים שם — הם יהיו זמינים לכל העמודים.

## כללי שבירת עמודים (דסקטופ בלבד)

מתוחמים ב-`html[data-variant="desktop"]` כדי שלא יחולו על המובייל (שם הם היו חותכים תוכן):
- כרטיסיות ויחידות אטומיות **לא נחתכות באמצע** (`break-inside:avoid`).
- כותרת תמיד נשארת עם התוכן שאחריה (`break-after:avoid`).
- אוויר בראש כל עמוד (שולי `@page`).
- בעמוד התהליך: חלקים בעלי רקע-גוון נסגרים/נפתחים בכל עמוד (`box-decoration-break:clone`).
- סקשן שגדול מעמוד שלם — מפצלים אותו לכמה כרטיסיות בגבולות טבעיים.
